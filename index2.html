Scalability summary table
The dilution effect: at larger scales, recovered tasks are spread across a much bigger pool of surviving VMs, so the per-VM load impact shrinks. FT-GA becomes most cost-efficient exactly where fault tolerance is needed most.

About This Project

Final Year Dissertation — Cloud Computing · Intelligent Scheduling · Fault Tolerance

Research gap

Most prior work treats fault tolerance and intelligent scheduling as separate concerns. This project unifies them in a single framework — GA optimises the schedule globally, then the fault tolerance layer handles VM failures reactively using a greedy minimum-load recovery policy, without re-running the expensive evolutionary search.

GA fitness function
f(chromosome) = α · makespan + β · imbalance
α = 0.7  (makespan weight)
β = 0.3  (balance weight)
makespan = maxv( Σ execTime(t,v) )
imbalance = maxv(load) − minv(load)
